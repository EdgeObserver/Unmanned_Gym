# 人脸识别模块设计说明文档

## 1. 概述

人脸识别模块是无人健身房系统的核心入口控制组件，采用Python + Java混合架构，结合深度学习人脸特征提取、向量相似度检索和消息队列异步处理，实现用户刷脸签到/签退的自动化管理。

## 2. 模块架构

### 2.1 技术栈

**Python服务**:
- **Web框架**: Flask
- **人脸识别**: DeepFace (Facenet模型)
- **向量检索**: FAISS (Facebook AI Similarity Search)
- **图像处理**: OpenCV, PIL
- **HTTP通信**: requests

**Java服务**:
- **框架**: Spring Boot + Spring Cloud
- **微服务通信**: OpenFeign
- **消息队列**: RabbitMQ (AMQP)
- **数据持久化**: MyBatis-Plus + MySQL

### 2.2 分层架构

```
┌──────────────────────────┐
│   Python Face Service    │  ← 人脸识别、特征提取、向量检索
├──────────────────────────┤
│   Java Controller Layer  │  ← REST API接口层
├──────────────────────────┤
│   Java Service Layer     │  ← 业务逻辑、订单验证
├──────────────────────────┤
│   Message Queue Layer    │  ← RabbitMQ事件分发
├──────────────────────────┤
│   Listener Layer         │  ← 事件消费、记录保存
├──────────────────────────┤
│   External Services      │  ← User/Order/Record微服务
└──────────────────────────┘
```

## 3. 核心组件说明

### 3.1 Python人脸识别服务 (FaceService.py)

#### Flask应用接口

##### `/py/face/build` (POST) - 构建人脸库
**功能**: 从图片目录批量提取人脸特征并构建FAISS索引

**流程**:
1. 遍历指定目录下所有图片(.jpg, .jpeg, .png)
2. 按文件名前缀聚类(如`zhangsan_0001.jpg` → `zhangsan`)
3. 对每人多张图片提取特征并计算平均向量
4. 使用DeepFace(Facenet模型)提取128维特征向量
5. 构建FAISS内积索引(IndexFlatIP)
6. 序列化保存到`bank.pk`文件

**关键代码**:
```python
def do_build(db_dir):
    # 按前缀聚类
    group = defaultdict(list)
    for p in img_paths:
        prefix = p.stem.split('_')[0]
        group[prefix].append(p)
    
    # 每人平均向量
    for prefix, paths in group.items():
        embs = [get_emb(img_path=p) for p in paths]
        center = np.mean(embs, axis=0)
        bank.append(center)
        labels_list.append(prefix)
    
    # FAISS索引
    faiss.normalize_L2(bank)
    index_new = faiss.IndexFlatIP(bank.shape[1])
    index_new.add(bank)
    
    # 持久化
    pickle.dump((index_new, labels_list), f)
```

##### `/py/face/start` (POST) - 启动识别服务
**功能**: 加载人脸库并启动后台摄像头识别线程

**流程**:
1. 检查服务是否已在运行
2. 从`bank.pk`加载FAISS索引和标签
3. 设置`is_running = True`
4. 创建daemon线程执行`background_loop()`
5. 打开摄像头开始实时识别

##### `/py/face/stop` (POST) - 停止识别服务
**功能**: 停止识别循环并释放资源

**流程**:
1. 设置`is_running = False`
2. 释放摄像头资源
3. 关闭OpenCV窗口

##### `/py/face/status` (GET) - 查询服务状态
**功能**: 返回当前服务运行状态

#### 人脸识别引擎

##### `get_emb(img_path, img_array)` - 特征提取
**功能**: 从图片中提取128维人脸特征向量

**步骤**:
1. 读取图片(cv2.imread或直接使用数组)
2. 使用DeepFace.extract_faces检测人脸(YuNet检测器)
3. 裁剪并resize到160x160
4. 调用DeepFace.represent(Facenet模型)提取特征
5. 返回128维embedding向量

##### `query(frame)` - 实时识别
**功能**: 在视频帧中识别人脸并匹配身份

**步骤**:
1. 调用`get_emb(img_array=frame)`提取当前帧特征
2. L2归一化特征向量
3. FAISS索引搜索最相似向量(`index.search(q, 1)`)
4. 计算相似度分数: `score = 1 - distance`
5. 返回用户ID和相似度

##### `background_loop(cam_url, interval)` - 识别主循环
**功能**: 持续从摄像头捕获帧并进行人脸识别

**核心逻辑**:
```python
while is_running:
    ret, frame = cap.read()
    
    # 识别
    uid, score = query(frame)
    
    # 阈值判断 + 冷却期控制
    if score > 0.4 and now - last_push >= push_gap:
        last_push = now
        # 异步推送识别结果到Java服务
        threading.Thread(target=async_push, args=(uid,)).start()
        
        # 获取后端返回的姓名和提示
        current_name = name_queue.get_nowait()
        current_msg = msg_queue.get_nowait()
        last_name_time = now
    
    # 显示超时清空
    if now - last_name_time > display_duration:
        current_name = ""
    
    # 绘制姓名到画面
    draw.text((10, 30), f"姓名：{current_name} {current_msg}")
    cv2.imshow("win", frame)
```

**特性**:
- **阈值过滤**: score > 0.4才认为识别成功
- **冷却期**: 3秒内不重复推送同一用户
- **显示持续时间**: 姓名显示2秒后自动消失
- **异步推送**: 避免阻塞主循环

##### `async_push(uid)` - 异步推送
**功能**: 将识别结果推送到Java服务

**流程**:
1. POST请求到Java服务的`/check-in`接口
2. 发送JSON: `{code: 200, msg: "success", data: uid}`
3. 接收Java返回的用户名和提示信息
4. 将结果放入队列供主循环显示

### 3.2 Java控制器层 (FaceRecognizeController.java)

#### 接口定义

##### `/check-in` (POST) - 签到的入口

**功能**: 处理用户签退请求

**流程**:
1. 调用`FaceRecognizeService.checkOutFaceId(id)`
2. 返回签退结果

### 3.3 Java服务层 (FaceRecognizeServiceImpl.java)

#### 核心业务逻辑

##### `checkInFaceId(id)` - 签到处理
**功能**: 验证用户资格并处理签到

**验证流程**:
1. **订单验证**:
   - 调用OrderFeignClient查询用户有效订单
   - 检查订单是否存在
   - 检查订单是否过期(`orderEndTime < today`)
   - 如果过期，更新订单状态为0并拒绝签到

2. **在场状态检查**:
   - 调用UserFeignClient查询用户信息
   - 检查`user.isInPlace()`是否为true
   - 如果已在场，拒绝重复刷脸

3. **签到处理**:
   - 设置`user.setInPlace(true)`
   - 更新用户到场状态
   - 计算剩余天数: `ChronoUnit.DAYS.between(today, endTime) + 1`

4. **发送MQ事件**:
   ```java
   FaceEventMessage message = new FaceEventMessage(
       id,
       "CHECK_IN",
       LocalDateTime.now(),
       user.getUsername()
   );
   rabbitTemplate.convertAndSend(
       "face.event.exchange",
       "face.checkin",
       message
   );
   ```

5. **返回结果**:
   - 成功: `"刷脸成功，剩余天数：X"`
   - 失败: `"您的订单已过期"` 或 `"禁止重复刷脸"`

##### `checkOutFaceId(id)` - 签退处理
**功能**: 处理用户签退

**流程**:
1. 查询用户信息
2. 检查`user.isInPlace()`是否为false
3. 如果不在场，拒绝重复签退
4. 设置`user.setInPlace(false)`
5. 更新用户状态
6. 发送签退MQ事件(`ROUTING_KEY_CHECKOUT`)
7. 返回"刷脸成功"

### 3.4 消息队列层 (RabbitMQ)

#### 交换机配置
- **交换机名称**: `face.event.exchange`

## 5. 关键技术细节

### 5.1 人脸特征提取

**模型**: Facenet (通过DeepFace封装)
- **输入尺寸**: 160x160 RGB
- **输出维度**: 128维浮点向量
- **检测器**: YuNet (轻量级人脸检测)
- **归一化**: L2归一化用于相似度计算

**特征提取代码**:
```python
faces = DeepFace.extract_faces(
    img, 
    detector_backend="yunet",
    enforce_detection=False
)
face_rgb = faces[0]["face"]
face = cv2.resize(face_rgb, (160, 160))
face = (face * 255).astype(np.uint8)
emb = DeepFace.represent(
    face, 
    model_name="Facenet",
    detector_backend="skip"
)[0]["embedding"]
```

### 5.2 FAISS向量检索

**索引类型**: IndexFlatIP (内积索引)
- **适用场景**: 小规模数据集(<1M向量)
- **相似度度量**: 内积(IP)，配合L2归一化等价于余弦相似度
- **搜索方式**: 暴力搜索(精确匹配)

**检索代码**:
```python
faiss.normalize_L2(bank)  # 建库时归一化
index = faiss.IndexFlatIP(128)
index.add(bank)

# 查询时
faiss.normalize_L2(query)
D, I = index.search(query, 1)  # top-1
score = 1 - D[0, 0]  # 转换为相似度
```

### 5.3 防重复机制

**双重防护**:
1. **Python端冷却期**: 3秒内不重复推送同一用户
2. **Java端状态检查**: 检查`user.inPlace`防止重复签到/签退

**显示控制**:
- 姓名显示2秒后自动消失
- 避免画面长期显示造成干扰

### 5.4 异步处理

**Python异步推送**:
```python
threading.Thread(
    target=async_push, 
    args=(uid,), 
    daemon=True
).start()
```
- 不阻塞主识别循环
- daemon线程随主进程退出

**Java MQ异步解耦**:
- 签到逻辑与记录保存分离
- 提高系统响应速度
- 增强可扩展性

## 6. 异常处理

### 6.1 Python端异常

| 异常类型 | 处理方式 |
|---------|---------|
| 人脸检测失败 | 跳过当前帧，继续下一帧 |
| 图片读取失败 | 打印错误，跳过该图片 |
| HTTP推送失败 | 捕获异常，打印堆栈 |
| 摄像头打开失败 | 重试5次，失败则退出线程 |

### 6.2 Java端异常

| 异常类型 | 处理方式 |
|---------|---------|
| 订单不存在 | 返回"订单已过期" |
| 订单已过期 | 更新订单状态，返回失败 |
| 重复刷脸 | 返回"禁止重复刷脸" |
| MQ发送失败 | 打印错误堆栈，不影响主流程 |
| Feign调用超时 | 抛出异常，返回500错误 |

## 7. 性能优化

### 7.1 Python端优化
- **特征缓存**: 人脸库一次性加载到内存
- **异步推送**: 不阻塞识别主循环
- **冷却期控制**: 减少无效HTTP请求
- **轻量检测器**: 使用YuNet而非MTCNN

### 7.2 Java端优化
- **Feign连接池**: 复用HTTP连接
- **MQ异步解耦**: 快速响应前端
- **事务控制**: 保证数据一致性
- **索引优化**: 用户表inPlace字段加索引

### 7.3 数据库优化
- **用户表索引**: `idx_in_place`加速状态查询
- **订单表索引**: `idx_user_end_time`加速有效期检查
- **记录表分区**: 按月分表存储签到记录
