# 人数检测模块设计说明文档

## 1. 概述

人数检测模块是无人健身房系统的安全监控组件，采用Python + Java混合架构，结合YOLOv5目标检测算法和实时WebSocket推送，实现对健身房内人数的实时监控和异常警报。该模块与人脸识别模块联动，通过对比"应有人数"和"实际检测人数"，及时发现并预警人员异常情况。

## 2. 模块架构

### 2.1 技术栈

**Python服务**:
- **Web框架**: Flask
- **目标检测**: YOLOv5 (Ultralytics)
- **图像处理**: OpenCV
- **视频流**: MJPEG (multipart/x-mixed-replace)

**Java服务**:
- **框架**: Spring Boot
- **定时任务**: @Scheduled
- **消息队列**: RabbitMQ
- **实时通信**: WebSocket (STOMP协议)
- **HTTP客户端**: Java HttpClient

### 2.2 分层架构

```
┌──────────────────────────┐
│ Python Detect Service    │  ← YOLO检测、人数统计、视频流
├──────────────────────────┤
│   Java Controller Layer  │  ← REST API接口层
├──────────────────────────┤
│   Java Service Layer     │  ← 定时轮询、状态对比
├──────────────────────────┤
│   Utility Layer          │  ← 人数计数器、防抖动
├──────────────────────────┤
│   Message Listener       │  ← 监听签到/签退事件
├──────────────────────────┤
│   WebSocket              │  ← 实时推送警报状态
└──────────────────────────┘
```

## 3. 核心组件说明

### 3.1 Python人数检测服务 (PersonDetectService.py)

#### Flask应用接口

##### `/py/person_count` (GET) - 获取当前人数
**功能**: 返回YOLO检测到的当前人数

**响应格式**:
```json
{
  "person_count": 5
}
```

##### `/py/video` (GET) - 视频流
**功能**: 提供MJPEG格式的实时视频流，带有人数标注

**响应类型**: `multipart/x-mixed-replace; boundary=frame`

**特性**:
- 实时显示检测框和置信度
- 左上角显示总人数
- 每3帧检测一次优化性能

#### 核心检测引擎

##### `detect(frame)` - 单帧检测
**功能**: 使用YOLOv5检测画面中的人数

**流程**:
1. 调用`model.predict(frame, conf=0.8, device="cpu")`
2. 过滤出person类别(COCO数据集class 0)
3. 统计人数: `person_count = len(persons)`
4. 在画面上绘制检测框和置信度
5. 显示总人数文本

**代码示例**:
```python
def detect(frame):
    # YOLO推理
    results = model.predict(frame, conf=0.8, device="cpu", verbose=False)
    
    # 只保留person类别
    persons = [d for d in results[0].boxes.data if int(d[5]) == 0]
    person_count = len(persons)
    
    # 绘制检测框
    for d in persons:
        x1, y1, x2, y2, conf, cls = d.tolist()
        cv2.rectangle(frame, (int(x1), int(y1)),
                      (int(x2), int(y2)), (0, 255, 0), 2)
        cv2.putText(frame, f"{conf:.2f}",
                    (int(x1), int(y1) - 5),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.6,
                    (0, 255, 0), 2)
    
    # 显示总人数
    cv2.putText(frame, f"Person: {person_count}",
                (10, 30), cv2.FONT_HERSHEY_SIMPLEX,
                1, (0, 0, 255), 2)
    
    return frame, person_count
```

##### `gen()` - 视频流生成器
**功能**: 持续捕获摄像头画面并生成MJPEG流

**优化策略**:
- **跳帧检测**: 每3帧检测一次(cnt % 3 == 1)
- **缓存机制**: 非检测帧复用上一帧结果
- **带宽优化**: 减少为原来的1/3

**代码逻辑**:
```python
def gen():
    cnt = 0
    cache = None
    
    while True:
        cnt += 1
        ret, frame = cap.read()
        
        if not ret:
            # 摄像头断开重连
            cap.release()
            time.sleep(2)
            cap = cv2.VideoCapture(1, cv2.CAP_DSHOW)
            continue
        
        # 每3帧检测一次
        if cnt % 3 == 1:
            frame, person_count = detect(frame)
            _, jpeg = cv2.imencode('.jpg', frame, 
                                   [cv2.IMWRITE_JPEG_QUALITY, 75])
            cache = jpeg
        
        # 跳过非检测帧
        if cache is None:
            continue
        
        # 输出MJPEG帧
        yield (b'--frame\r\n'
               b'Content-Type: image/jpeg\r\n\r\n' +
               cache.tobytes() + b'\r\n')
```

**配置参数**:
- **模型**: yolov5n.pt (nano版本，轻量快速)
- **置信度阈值**: 0.8
- **设备**: CPU
- **摄像头**: cv2.VideoCapture(1)
- **JPEG质量**: 75%

### 3.3 Java服务层 (CountServiceImpl.java)

#### 核心业务逻辑

##### `getCurrentCount()` - 获取应有人数
**功能**: 从PersonCountHolder获取当前应有人数

**返回**: ResultMsg包含current值

##### `arrive()` - 人员到达
**功能**: 应有人数+1

**实现**:
```java
personCountHolder.setCurrent(personCountHolder.getCurrent() + 1);
```

##### `leave()` - 人员离开
**功能**: 应有人数-1，确保不为负数

**实现**:
```java
personCountHolder.setCurrent(
    max(0, personCountHolder.getCurrent() - 1)
);
```

##### `checkCount(fresh)` - 人数对比检测
**功能**: 对比实际检测人数和应有人数

**流程**:
1. 调用`personCountHolder.checkCount(fresh)`
2. 根据返回结果封装ResultMsg
3. code=200表示正常，code=500表示异常

##### `pushColor()` - 定时推送颜色状态 ⭐核心方法
**功能**: 每500ms轮询Python服务，对比人数并推送警报状态

**执行频率**: `@Scheduled(fixedDelay = 500)`

**完整流程**:
```java
@Scheduled(fixedDelay = 500)
public void pushColor() {
    try {
        // 1. HTTP请求Python服务获取实际人数
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("http://localhost:" + port + "/py/person_count"))
            .GET()
            .timeout(Duration.ofSeconds(30))
            .build();
        
        HttpResponse<String> response = client.send(request,
            HttpResponse.BodyHandlers.ofString());
        
        // 2. 解析JSON获取人数
        ObjectMapper mapper = new ObjectMapper();
        JsonNode rootNode = mapper.readTree(response.body());
        int fresh = rootNode.get("person_count").asInt();
        
        // 3. 对比检测
        ResultMsg<Integer> result = checkCount(fresh);
        
        // 4. 确定颜色
        String color = result.getCode() == 500 ? "red" : "green";
        
        // 5. 变化时才推送(避免冗余)
        if (!color.equals(lastPushColor)) {
            lastPushColor = color;
            
            // 6. WebSocket广播
            ws.convertAndSend("/topic/color", Map.of("color", color));
        }
    } catch (IOException | InterruptedException e) {
        e.printStackTrace();
    }
}
```

**关键特性**:
- **定时轮询**: 500ms间隔平衡实时性和性能
- **变化检测**: 只在颜色变化时推送，减少网络负载
- **超时控制**: HTTP请求30秒超时
- **异常处理**: 捕获IO异常避免中断定时任务

### 3.4 工具类层 (PersonCountHolder.java)

#### 核心数据结构

**成员变量**:
```java
private volatile int current = 0;      // 当前应有人数
private final int threshold = 1;       // 触发阈值(连续几次超标)
private final AtomicInteger counter = new AtomicInteger(0); // 计数器
```

#### `checkCount(fresh)` - 人数对比逻辑
**功能**: 对比实际人数和应有人数，判断是否触发警报

**逻辑流程**:
```java
public ResultMsg<Integer> checkCount(int fresh) {
    if (fresh > current) {
        // 实际人数 > 应有人数
        int newCounter = counter.incrementAndGet();
        
        if (newCounter >= threshold) {
            // 达到阈值，触发警报
            return ResultMsg.fail(current, "人数异常");
        }
    } else {
        // 人数正常或偏少，重置计数器
        counter.set(0);
    }
    
    return ResultMsg.success(current, "人数正常");
}
```

**防抖动机制**:
- **原子计数器**: 使用AtomicInteger保证线程安全
- **阈值判断**: 连续threshold次超标才触发(当前设为1便于测试)
- **自动重置**: 一旦恢复正常立即清零计数器

**设计目的**:
- 避免因瞬时检测误差导致误报
- 支持调整灵敏度(threshold参数)
- 线程安全支持并发访问

### 3.5 消息监听层 (CountEventListener.java)

#### `handleCheckIn(message)` - 签到事件监听
**队列**: `face.checkin.queue`

**功能**: 监听到签到事件后，应有人数+1

**流程**:
1. 接收RabbitMQ消息
2. 反序列化为FaceEventMessage
3. 调用`countService.arrive()`
4. 打印日志显示更新后的应有人数

**代码**:
```java
@RabbitListener(queues = "face.checkin.queue")
public void handleCheckIn(Message message) {
    String body = new String(message.getBody());
    FaceEventMessage event = objectMapper.readValue(body, FaceEventMessage.class);
    
    countService.arrive();  // current++
    
    int currentCount = countService.getCurrentCount().getData();
    System.out.println("[RabbitMQ] ✅ 签到成功，当前应有人数: " + currentCount);
}
```

#### `handleCheckOut(message)` - 签退事件监听
**队列**: `face.checkout.queue`

**功能**: 监听到签退事件后，应有人数-1

**流程**:
1. 接收RabbitMQ消息
2. 反序列化为FaceEventMessage
3. 调用`countService.leave()`
4. 打印日志显示更新后的应有人数

## 5. 关键技术细节

### 5.1 YOLOv5检测

**模型选择**: yolov5n.pt (nano版本)
- **优点**: 速度快，适合CPU实时检测
- **缺点**: 精度略低于大模型
- **适用场景**: 人数统计不需要极高精度

**检测参数**:
- **置信度阈值**: 0.8 (高阈值保证准确性)
- **设备**: CPU (无需GPU)
- **类别**: 只检测person (COCO class 0)

**性能优化**:
- 每3帧检测一次
- 降低分辨率(可选)
- 使用nano模型

### 5.2 防抖动机制

**问题**: 瞬时检测误差可能导致误报

**解决方案**:
1. **原子计数器**: 记录连续超标次数
2. **阈值判断**: 达到threshold次才触发
3. **自动重置**: 恢复正常立即清零

**配置建议**:
- 测试环境: threshold=1 (快速验证)
- 生产环境: threshold=3~5 (更稳定)

### 5.3 WebSocket实时推送

**协议**: STOMP over WebSocket

**主题**: `/topic/color`

**消息格式**:
```json
{
  "color": "red"  // 或 "green"
}
```

**前端订阅**:
```javascript
const socket = new SockJS('/ws');
const stompClient = Stomp.over(socket);

stompClient.connect({}, function() {
    stompClient.subscribe('/topic/color', function(message) {
        const data = JSON.parse(message.body);
        if (data.color === 'red') {
            // 显示红色警报
            showAlarm();
        } else {
            // 显示绿色正常
            hideAlarm();
        }
    });
});
```

### 5.4 MJPEG视频流

**帧格式**:

## 6. 异常处理

### 6.1 Python端异常

| 异常类型 | 处理方式 |
|---------|---------|
| 摄像头断开 | 释放并重连(延时2秒) |
| YOLO推理失败 | 跳过当前帧，继续下一帧 |
| HTTP请求超时 | 返回错误，不影响下次请求 |

### 6.2 Java端异常

| 异常类型 | 处理方式 |
|---------|---------|
| HTTP连接失败 | 捕获异常，打印堆栈，继续下次轮询 |
| JSON解析失败 | 捕获异常，跳过本次检测 |
| WebSocket断开 | 自动重连机制 |
| RabbitMQ连接失败 | Spring自动重试 |

## 7. 性能优化

### 7.1 Python端优化
- **跳帧检测**: 每3帧检测一次，降低CPU负载67%
- **轻量模型**: 使用yolov5n而非yolov5l/x
- **JPEG压缩**: 质量75%平衡清晰度和带宽
- **缓存机制**: 非检测帧复用结果

### 7.2 Java端优化
- **变化检测**: 只在颜色变化时推送WebSocket
- **合理轮询间隔**: 500ms平衡实时性和负载
- **HTTP超时**: 30秒防止长时间阻塞
- **异步处理**: RabbitMQ监听异步不阻塞主线程

### 7.3 网络优化
- **MJPEG流**: 比WebRTC简单，延迟可接受
- **WebSocket**: 长连接避免频繁握手
- **最小化消息**: 只推送color字段
